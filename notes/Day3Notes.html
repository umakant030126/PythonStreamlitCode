Lesson 01

The Big Picture — Why Layout Matters 🗺️

On Day 1 and Day 2, every element we added went one below the other — like a long vertical list. That is fine for a simple script, but real apps need structure.

🏠 Analogy — Building a House
Imagine your Streamlit app is a house. So far we have been dumping furniture in one long hallway — bed, then sofa, then table, all in a row. Today we learn to create rooms. A living room (main area), a kitchen (sidebar), drawers (expanders), and floors (tabs). Each piece of furniture goes in the right room!

Streamlit gives us these layout tools. Here is a quick overview before we dive deep into each:

📐
st.columns()
Split the page into side-by-side columns. Like newspaper columns or a CSS flexbox row.
📌
st.sidebar
A fixed left panel for filters, settings, and navigation — always visible.
🔽
st.expander()
A collapsible section. Users click to open/close it. Good for optional details.
🗂️
st.tabs()
Tabbed navigation. Like browser tabs. Great for multi-section apps.
📦
st.container()
An invisible box to group elements together logically.
⬜
st.empty()
A placeholder that can be overwritten later — for live-updating content.
Lesson 02

st.columns() — Side-by-Side Layout 📐

The most used layout tool. st.columns() splits the page into side-by-side vertical sections. Think of a newspaper — it has multiple columns of text next to each other.

📰 Analogy — Newspaper Columns
A newspaper has 3–4 columns of text side by side. st.columns(3) does exactly the same for your Streamlit app — creates 3 equal-width columns. Whatever you put inside each column appears next to the other columns, not below them.
Visual — st.columns(3) layout
← Full page width (before columns) →
Column 1
col1 content
Column 2
col2 content
Column 3
col3 content
← Full page width (after columns) →
python — columns_demo.py
import streamlit as st

st.title("Columns Demo")

# ── EQUAL columns ─────────────────────────────────────────
# st.columns(3) creates 3 equal-width columns
# It returns 3 column objects — we unpack them into variables
col1, col2, col3 = st.columns(3)

# Use the "with" keyword to put content INSIDE a column
with col1:
    st.header("Column 1")
    st.write("I am in the first column.")
    st.button("Button in Col 1")

with col2:
    st.header("Column 2")
    st.write("I am in the second column.")
    st.slider("A slider:", 0, 100, 50)

with col3:
    st.header("Column 3")
    st.write("I am in the third column.")
    st.selectbox("Pick:", ["A", "B", "C"])

st.divider()

# ── UNEQUAL columns using a list ──────────────────────────
# [2, 1] means left column is TWICE as wide as right column
# [1, 3, 1] means middle is 3x wider than the sides
left, right = st.columns([2, 1])

with left:
    st.subheader("Main Content (2x wide)")
    st.write("This column gets 2/3 of the page width.")

with right:
    st.subheader("Side Panel (1x)")
    st.write("This gets 1/3 of the page.")

st.divider()

# ── GAP parameter — adds space BETWEEN columns ────────────
c1, c2 = st.columns(2, gap="large")
# gap options: "small" (default), "medium", "large"

with c1:
    st.write("Left side with large gap")
with c2:
    st.write("Right side with large gap")
UsageWhat it createsExample
st.columns(3)3 equal-width columnsThree equal panels
st.columns([2, 1])Left is 2x wider than rightMain content + small panel
st.columns([1, 2, 1])Middle is 2x wider than sidesCentred hero layout
st.columns(2, gap="large")2 columns with big gap between themBreathing room between panels
⚠️ Critical Rule — Always use "with"
To put content inside a column, you must use the with col1: syntax. If you call st.write() outside a with block after creating columns, it goes below the columns on the full page, not inside any column!
💡 Real-world use cases for columns
Showing 3 KPI cards side by side (like a dashboard) · Putting a chart on the left and a filter on the right · Displaying before/after comparison · Two-column form layout (name | email)
Lesson 03

st.sidebar — The Left Panel 📌

The sidebar is a collapsible left panel that appears beside your main content. It's the perfect place for filters, settings, navigation, and controls — things the user adjusts but doesn't want cluttering the main view.

📊 Analogy — Excel's Ribbon/Filter Panel
Think of it like the filter panel in Excel or the settings panel in any app. The main area shows results. The sidebar has the controls. User adjusts sidebar → main area updates automatically!
Visual — Sidebar layout
SIDEBAR
Filters
Settings
Navigation
MAIN CONTENT AREA
Results, Charts, Tables, Output
python — sidebar_demo.py
import streamlit as st

st.title("Sidebar Demo")

# ── METHOD 1: Using "with st.sidebar:" block ───────────────
# Everything inside this block goes into the sidebar
with st.sidebar:
    st.header("⚙️ Filters")
    st.divider()

    project = st.selectbox(
        "Select Project:",
        ["ICAPS", "IBSG", "Lumen Sky"]
    )

    env = st.radio(
        "Environment:",
        ["DEV", "QA", "STAGING", "PROD"]
    )

    show_passed = st.checkbox("Show Passed Tests", value=True)
    show_failed = st.checkbox("Show Failed Tests", value=True)

    st.divider()
    st.caption("QA Dashboard v1.0 | Umakant Pandey")

# ── MAIN CONTENT uses sidebar values ──────────────────────
# The variables (project, env) are available here too!
st.subheader(f"Results for: {project} ({env})")
st.write(f"Showing passed: {show_passed}")
st.write(f"Showing failed: {show_failed}")

st.divider()

# ── METHOD 2: Direct dot notation (alternative style) ─────
# You can also call st.sidebar.widget() directly
sprint = st.sidebar.number_input("Sprint number:", min_value=1, value=10)
st.write(f"Currently viewing Sprint {sprint}")
📘 Two ways to use sidebar
Method 1 — with block: with st.sidebar: then indent all sidebar code below.
Method 2 — dot notation: st.sidebar.selectbox(...) — call any widget directly on sidebar.

Both work the same. Method 1 is cleaner when you have many sidebar items. Method 2 is quick for one or two items.
💡 Pro tip — Sidebar title and image
You can add a logo or title in the sidebar too:
st.sidebar.title("My App")
st.sidebar.image("logo.png", width=150)
This gives your app a professional branded look!
Lesson 04

st.expander() — Collapsible Sections 🔽

An expander is a section that starts collapsed (hidden). The user clicks a title/arrow to open it and see the content inside. Perfect for optional details, raw data, debug info, or long explanations you don't want to clutter the main view.

📬 Analogy — Accordion File Folder
Think of a filing cabinet with accordion folders. Each folder is labelled on the outside. You can see all labels at once. When you need to read something, you pull open just that folder. st.expander() works the same — user sees all section titles, opens only what they need.
python — expander_demo.py
import streamlit as st

st.title("Expander Demo")
st.write("Main visible content is here. Expand sections below for details.")

# ── Basic expander — closed by default ────────────────────
with st.expander("📋 Click to see Test Steps"):
    st.write("Step 1: Login to application")
    st.write("Step 2: Navigate to dashboard")
    st.write("Step 3: Verify KPI cards display correctly")
    st.write("Step 4: Logout")

# ── Expander open by default using expanded=True ──────────
with st.expander("📊 Test Results Summary", expanded=True):
    st.success("✅ 47 tests Passed")
    st.error("❌ 3 tests Failed")
    st.warning("⚠️ 2 tests Blocked")

# ── Multiple expanders act like an accordion ───────────────
bugs = [
    {"id": "BUG-001", "title": "Login fails on Chrome", "severity": "High"},
    {"id": "BUG-002", "title": "Dashboard timeout",    "severity": "Medium"},
    {"id": "BUG-003", "title": "Export button missing", "severity": "Low"},
]

st.subheader("🐛 Bug Details")
for bug in bugs:
    with st.expander(f"{bug['id']} — {bug['title']}"):
        st.write(f"**Severity:** {bug['severity']}")
        st.write("**Status:** Open")
        st.write("**Assigned to:** Umakant Pandey")
        st.text_area("Add comment:", key=bug["id"])
📘 The "key" parameter
Notice key=bug["id"] in the loop above. When you create multiple same-type widgets in a loop, Streamlit needs a unique key for each one to tell them apart. Always set a unique key when creating widgets inside loops!
Lesson 05

st.tabs() — Tabbed Navigation 🗂️

Tabs are like browser tabs or the tabs in an Excel workbook. Users click a tab name to switch between completely different views — without leaving the page or reloading.

Visual — st.tabs() layout
Tab 1
Tab 2 ← Active
Tab 3
Content of Tab 2 is shown here
python — tabs_demo.py
import streamlit as st

st.title("🗂️ QA Report — Tabbed View")

# st.tabs() takes a list of tab names
# Returns a list of tab objects — unpack them!
tab1, tab2, tab3, tab4 = st.tabs([
    "📊 Summary",
    "🐛 Bug Report",
    "✅ Test Cases",
    "⚙️ Settings"
])

with tab1:
    st.header("Test Execution Summary")
    st.success("Total Tests: 52 | Passed: 47 | Failed: 3 | Blocked: 2")
    st.write("Overall pass rate: 90.4%")

with tab2:
    st.header("Bug Report")
    st.error("3 open bugs found in this sprint")
    st.write("• BUG-001: Login fails on Chrome — HIGH")
    st.write("• BUG-002: Dashboard timeout — MEDIUM")
    st.write("• BUG-003: Export button missing — LOW")

with tab3:
    st.header("Test Case List")
    tc_data = {
        "Test ID": ["TC-001", "TC-002", "TC-003"],
        "Name":    ["Login Test", "Logout Test", "Dashboard Load"],
        "Status":  ["Pass", "Pass", "Fail"]
    }
    st.write(tc_data)   # st.write shows dicts as a nice table!

with tab4:
    st.header("Settings")
    st.selectbox("Theme:", ["Light", "Dark"])
    st.checkbox("Email notifications", value=True)
    st.text_input("Your email:")
💡 Tabs vs Expanders — When to use which?
Use Tabs when content sections are alternatives — user wants to see ONE section at a time (like Summary OR Bug Report, not both).
Use Expanders when content is supplementary — user might want to see the main content AND open a detail section simultaneously.
Lesson 06

st.container() — Grouping Elements 📦

A container is an invisible box that groups elements together. You can't see it on screen, but it helps you organise code and control the order elements appear — especially useful when you want to add content in a different order than it appears.

python — container_demo.py
import streamlit as st

st.title("Container Demo")

# ── Use case 1: Logical grouping ──────────────────────────
with st.container():
    st.subheader("Section A — Inputs")
    st.text_input("Name:")
    st.number_input("Age:", value=25)

with st.container():
    st.subheader("Section B — Options")
    st.selectbox("Role:", ["QA Engineer", "Developer"])
    st.checkbox("Active employee")

st.divider()

# ── Use case 2: Add border around a container ─────────────
# border=True draws a visible box around the container
with st.container(border=True):
    st.subheader("🧪 Test Run #42")
    st.write("Environment: QA")
    st.write("Status: In Progress")
    st.progress(0.65)  # Shows a 65% filled progress bar

st.divider()

# ── Use case 3: Out-of-order writing ─────────────────────
# Create a container FIRST, then fill it LATER in the code!
result_box = st.container()  # Reserve the spot at the top

st.write("This line appears BELOW the result box on screen.")
name = st.text_input("Type your name:")

# Write INTO the container (appears at top, even though code is at bottom!)
with result_box:
    st.success(f"👋 Hello, {name or 'stranger'}!")
📘 The Out-of-Order Magic
The most powerful use of st.container() is writing content out of order. Create the container early in the code to reserve a spot on the page, then fill it later. The result appears at the top even though the code writing it is at the bottom. Very useful for showing dynamic results!
Lesson 07

st.empty() — Placeholder That Updates ⬜

st.empty() creates a single-element placeholder. You can overwrite it as many times as you like — the new content replaces the old one in the same spot. Great for live updates, countdowns, or status messages that change.

python — empty_demo.py
import streamlit as st
import time

st.title("Empty Placeholder Demo")

# ── Use case 1: Dynamic status message ────────────────────
status = st.empty()   # Reserve a spot
status.info("⏳ Ready to start...")  # Initial state

if st.button("Run Test Suite"):
    status.warning("🔄 Tests running...")
    time.sleep(2)   # Simulate test execution
    status.success("✅ All tests complete!")

st.divider()

# ── Use case 2: Countdown timer ───────────────────────────
if st.button("Start 5-second countdown"):
    placeholder = st.empty()
    for i in range(5, 0, -1):
        placeholder.metric("Countdown", f"{i} seconds")
        time.sleep(1)
    placeholder.success("🚀 Launch!")

st.divider()

# ── Use case 3: Clear a message ───────────────────────────
msg = st.empty()
if st.button("Show message"):
    msg.success("Hello!")
if st.button("Clear message"):
    msg.empty()  # Call .empty() on it to CLEAR the content
💡 container vs empty — Key difference
st.container() can hold multiple elements. You keep adding to it.
st.empty() holds exactly ONE element. Each new write replaces the previous one.
Lesson 08

st.set_page_config() — App Settings 🎛️

This is the first function you should call in every Streamlit app. It sets the browser tab title, the favicon icon, the layout width, and the sidebar default state. It must be the very first Streamlit command in your script.

python — always put this FIRST in every app!
import streamlit as st

# ── MUST be the FIRST st.* call in your script ─────────────
st.set_page_config(
    page_title="QA Dashboard",          # Browser tab title
    page_icon="🛡️",                    # Favicon — emoji or image path
    layout="wide",                      # "centered" (default) or "wide"
    initial_sidebar_state="expanded",  # "expanded" or "collapsed"
    menu_items={                         # Custom hamburger menu items
        'Get Help': 'https://streamlit.io/docs',
        'About': "QA Dashboard built by Umakant Pandey"
    }
)

# Now the rest of your app code goes below
st.title("🛡️ QA Dashboard")
ParameterWhat it controlsOptions
page_titleText shown in browser tabAny string
page_iconFavicon in browser tabEmoji string or image path
layoutPage width"centered" (narrow, default) or "wide" (full width)
initial_sidebar_stateSidebar open or closed on load"expanded" or "collapsed"
menu_itemsCustom links in the ☰ hamburger menuDict with keys: Get Help, Report a bug, About
⚠️ Rule — First Line Always
st.set_page_config() must be the very first Streamlit function call in your script. If you call st.title() or any other st.* function before it, you will get an error! Always put it right after your imports.
Lesson 09

st.metric() — KPI Cards 📊

Metric cards are those large-number display blocks you see on dashboards — like "Total Sales: ₹1.2M ▲5%". st.metric() creates exactly that. Used almost always inside columns to create side-by-side KPI cards.

python — metric_demo.py
import streamlit as st

st.title("📊 QA KPI Dashboard")

# ── Three metric cards side by side ───────────────────────
col1, col2, col3, col4 = st.columns(4)

with col1:
    st.metric(
        label="Total Tests",
        value="52",
        delta="+8 from last sprint"   # Green arrow up
    )

with col2:
    st.metric(
        label="Tests Passed",
        value="47",
        delta="+5"   # Positive = green arrow ▲
    )

with col3:
    st.metric(
        label="Tests Failed",
        value="3",
        delta="-2",              # Negative = red arrow ▼
        delta_color="inverse"   # "inverse" = down is GOOD here!
    )

with col4:
    st.metric(
        label="Pass Rate",
        value="90.4%",
        delta="2.1%",
        delta_color="normal"    # "normal" (default), "inverse", "off"
    )
ParameterWhat it shows
labelSmall text above the big number (e.g. "Total Tests")
valueThe big number/text displayed prominently
deltaChange indicator — shown with green ▲ or red ▼ arrow
delta_color="normal"Positive = green, negative = red (default)
delta_color="inverse"Positive = red, negative = green (use when DOWN is good, like "bugs fixed")
delta_color="off"Shows delta in grey with no colour

🏗️ Mini Project — Full QA Dashboard

Now let's combine everything from Day 1, 2, and 3 into one real QA dashboard. This is the kind of app you can actually use at work or show to your team!

python — qa_dashboard.py — your Day 3 project!
import streamlit as st
import datetime

# ── Page Config — always first! ───────────────────────────
st.set_page_config(
    page_title="QA Dashboard",
    page_icon="🛡️",
    layout="wide",
    initial_sidebar_state="expanded"
)

# ── SIDEBAR — Filters ─────────────────────────────────────
with st.sidebar:
    st.title("🛡️ QA Dashboard")
    st.caption("By Umakant Pandey")
    st.divider()

    project = st.selectbox(
        "Project:",
        ["ICAPS (Citi Bank)", "IBSG (Citi Bank)", "Lumen Sky"]
    )
    sprint = st.number_input("Sprint:", min_value=1, value=42)
    env = st.radio("Environment:",
                   ["DEV", "QA", "STAGING"], horizontal=True)
    date_range = st.date_input(
        "Test Window:",
        value=(datetime.date.today(), datetime.date.today())
    )
    st.divider()
    tester = st.text_input("Tester Name:", value="Umakant Pandey")

# ── MAIN AREA ─────────────────────────────────────────────
st.title(f"📊 {project} — Sprint {sprint} Report")
st.caption(f"Environment: {env}  |  Tester: {tester}")
st.divider()

# ── KPI CARDS row ─────────────────────────────────────────
k1, k2, k3, k4 = st.columns(4)
with k1: st.metric("Total Tests",  "52",  "+8")
with k2: st.metric("Passed",       "47",  "+5")
with k3: st.metric("Failed",       "3",   "-2", delta_color="inverse")
with k4: st.metric("Pass Rate",    "90.4%", "↑2.1%")

st.divider()

# ── TABBED SECTIONS ───────────────────────────────────────
t1, t2, t3 = st.tabs(["✅ Test Cases", "🐛 Bugs", "📝 Add Report"])

with t1:
    left, right = st.columns([3, 1])
    with left:
        st.subheader("Test Case Results")
        data = {
            "TC ID":  ["TC-001","TC-002","TC-003","TC-004"],
            "Name":   ["Login","Logout","Dashboard","Export"],
            "Result": ["Pass","Pass","Fail","Pass"],
            "Time(s)":[2.3,1.1,5.8,3.2]
        }
        st.write(data)
    with right:
        st.subheader("Filter")
        st.radio("Show:", ["All","Pass","Fail"])

with t2:
    st.subheader("Open Bugs")
    bugs = [
        ("BUG-001", "Login fails on Chrome", "🔴 High"),
        ("BUG-002", "Dashboard timeout",    "🟡 Medium"),
        ("BUG-003", "Export button missing", "🟢 Low"),
    ]
    for bid, btitle, bsev in bugs:
        with st.expander(f"{bid}: {btitle} — {bsev}"):
            st.write(f"**Severity:** {bsev}")
            st.write("**Assigned to:** Umakant Pandey")
            st.text_area("Comment:", key=bid)

with t3:
    st.subheader("Add Test Report Entry")
    c1, c2 = st.columns(2)
    with c1:
        st.text_input("Test Case ID:")
        st.selectbox("Result:", ["Pass","Fail","Blocked","Skip"])
    with c2:
        st.text_input("Test Case Name:")
        st.number_input("Execution time (s):", min_value=0.0, step=0.1)
    st.text_area("Notes / Error Description:")
    if st.button("📥 Submit Entry", type="primary"):
        st.success("✅ Report entry submitted successfully!")

This one file combines: set_page_config + sidebar + columns + metric + tabs + expander + widgets from Day 2. Run it and you have a real professional QA dashboard. Change the data to match your actual projects!
