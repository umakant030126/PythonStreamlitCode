Lesson 01

The KEY Concept — What Widgets Return 🔑

Before learning any individual widget, you must understand this one golden rule. Everything today builds on it.

🎰 Analogy — The Vending Machine
A vending machine is a widget. You press a button (interact), and it gives you something back — a snack (a value). In Streamlit, every widget is like a vending machine. You place it on the page, the user interacts, and it returns a Python value to your script. You capture that value in a variable, then use it in your code!

Every single widget follows this same pattern. Study it — this is the most important line in all of Streamlit:

python — the universal widget pattern
# PATTERN:  variable = st.widget_name("label shown to user")

user_value = st.some_widget("Label shown to user")

# Now "user_value" holds whatever the user picked / typed / selected
# Use it in any Python logic below!

st.write("You chose:", user_value)
📘 Remember from Day 1
When a user interacts with a widget, Streamlit reruns the whole script. But the widget remembers the user's last choice and returns it again automatically. So your variable always holds the latest value!
Lesson 02

st.button() — Click Me! 🔘

The most basic widget. A clickable button. Returns True when clicked. Returns False when not clicked.

🚪 Analogy — A Doorbell
A doorbell returns a signal only when someone presses it. The moment they stop, the signal is gone. st.button() works the same way — it is True only during the exact rerun triggered by the click. Next rerun, it goes back to False.
python — button_demo.py
import streamlit as st

st.title("Button Demo")

# Basic button — True when clicked, False otherwise
clicked = st.button("Click Me!")

if clicked:
    st.write("🎉 You clicked it! clicked =", clicked)
else:
    st.write("Button not clicked. clicked =", clicked)

st.divider()

# Show content only when clicked
if st.button("Reveal My Name"):
    st.success("My name is Umakant Pandey! 👋")

st.divider()

# type="primary" makes it a bold coloured button
if st.button("Submit Report", type="primary"):
    st.success("✅ Report submitted successfully!")

# disabled=True makes button unclickable (greyed out)
st.button("I am disabled", disabled=True)
ParameterWhat it doesExample
labelText shown on the button (required)"Click Me"
type"secondary" (grey, default) or "primary" (blue/bold)type="primary"
disabledIf True, button is greyed out and unclickabledisabled=True
helpTooltip shown on hoverhelp="Submits the form"
ReturnsTrue when clicked, False otherwise
💡 New functions sneak peek
st.success("text") → green box  |  st.error("text") → red box  |  st.warning("text") → yellow box  |  st.info("text") → blue box. We will cover these fully on Day 3!
Lesson 03

st.text_input() — Single Line Text Box ⌨️

Creates a one-line input box where the user types text. Like a search bar or a login field. Returns a Python str.

python — text_input_demo.py
import streamlit as st

st.title("Text Input Demo")

# Basic — returns "" when empty, or whatever user typed
name = st.text_input("Enter your name:")

if name:          # if name is NOT empty string
    st.write(f"Hello, {name}! Welcome to Streamlit! 👋")
else:
    st.write("Please type your name above.")

st.divider()

# With value (pre-filled default) and placeholder (hint text)
city = st.text_input(
    "Your city:",
    value="New Delhi",           # Already filled on load
    placeholder="Type city name...", # Grey hint text
    max_chars=50                 # Can't type more than 50 chars
)
st.write("City entered:", city)

st.divider()

# Password field — hides what the user types!
password = st.text_input("Enter password:", type="password")
if password:
    st.write("Password length:", len(password), "characters")
ParameterWhat it doesDefault
labelText shown above the input box (required)—
valuePre-filled text when app loads""
placeholderGrey hint text that disappears when user types""
type"default" (visible) or "password" (hidden dots)"default"
max_charsMaximum characters the user can typeNone
ReturnsA str — whatever the user typed
Lesson 04

st.text_area() — Multi-line Text Box 📄

st.text_input() = one line (search bar). st.text_area() = a big box for long writing — like a comment field, notepad, or message form.

python — textarea_demo.py
import streamlit as st

st.title("Text Area Demo")

feedback = st.text_area(
    "Write your feedback:",
    placeholder="Tell us what you think...",
    height=150   # Height of the text box in pixels
)

if feedback:
    word_count = len(feedback.split())
    char_count = len(feedback)
    st.write(f"Words: {word_count}  |  Characters: {char_count}")
    st.info(feedback)  # Show feedback in a blue box
📘 text_input vs text_area — Quick rule
Use st.text_input() for short answers: name, email, search query, one word.
Use st.text_area() for long answers: comments, bug descriptions, messages, test notes.
Lesson 05

st.number_input() — Number Field 🔢

A text box built specifically for numbers. Has up/down arrows. Prevents the user from typing letters. Returns an int or float.

python — number_input_demo.py
import streamlit as st

st.title("Number Input Demo")

# Integer number
age = st.number_input(
    "Enter your age:",
    min_value=1,       # Can't go below 1
    max_value=120,     # Can't go above 120
    value=25,          # Default value
    step=1             # Each arrow click adds/subtracts 1
)
st.write(f"You are {age} years old.")

st.divider()

# Float (decimal) number
salary = st.number_input(
    "Monthly salary (INR):",
    min_value=0.0,
    value=50000.0,
    step=1000.0,
    format="%.2f"    # Show exactly 2 decimal places
)
annual = salary * 12
st.write(f"Annual CTC: ₹{annual:,.0f}")
ParameterWhat it does
min_valueLowest number allowed
max_valueHighest number allowed
valueDefault number shown at start
stepHow much each arrow click changes. Use 1 for integer, 0.5 for float
formatDisplay format string. "%.2f" = 2 decimal places
Returnsint if step is integer, float if step is decimal
Lesson 06

st.slider() — Drag to Pick a Value 🎚️

A draggable bar. User slides left or right to pick a value. More visual and satisfying than a number input.

🔊 Analogy — Volume Dial
Think of a speaker's volume dial. You drag it between Min and Max to set your level. st.slider() is exactly that — a dial for any numeric value in your app.
python — slider_demo.py
import streamlit as st

st.title("Slider Demo")

# Single value slider
# Arguments in order: label, min, max, default value
age = st.slider("Select your age:", 1, 100, 25)
st.write(f"Age: {age}")

st.divider()

# Range slider — pass a TUPLE as default value!
# Returns a tuple (low, high) — two handles appear
salary_range = st.slider(
    "Expected salary range (₹K):",
    min_value=10,
    max_value=200,
    value=(30, 80)   # Tuple = range slider with 2 handles!
)
st.write(f"Looking for: ₹{salary_range[0]}K to ₹{salary_range[1]}K")

st.divider()

# Float slider with step
rating = st.slider("Rate this course:", 0.0, 5.0, 4.5, 0.5)
#                   label,             min,  max,  default, step
st.write(f"Rating: {'⭐' * int(rating)} ({rating}/5)")
⚠️ Range Slider Trick
Pass a tuple as the value — value=(30, 80) — and the slider becomes a range picker with two handles. It then returns a tuple (low, high). Access the values with range[0] (low) and range[1] (high).
Lesson 07

st.selectbox() — Dropdown Menu 📋

A dropdown list. User clicks to open it, picks one option. Returns the selected item.

python — selectbox_demo.py
import streamlit as st

st.title("Selectbox Demo")

# Pass a list of options
dept = st.selectbox(
    "Choose your department:",
    ["QA Testing", "Development", "DevOps", "Management"]
)
st.write(f"You selected: {dept}")

st.divider()

# index=0 means first item is default
lang = st.selectbox(
    "Favourite programming language:",
    ["Python", "Java", "JavaScript", "SQL"],
    index=0
)

# Use selection in Python logic — like a switch/match!
tips = {
    "Python":     "Great for data, automation, and QA scripting!",
    "Java":       "Your Selenium scripts run on this!",
    "JavaScript": "King of the browser — used in web testing.",
    "SQL":        "Essential for database testing — you already know this!"
}
st.info(tips[lang])
ParameterWhat it does
labelQuestion shown above dropdown
optionsA Python list of choices
indexDefault selected item (0 = first item)
ReturnsThe selected item — same type as items in your list
Lesson 08

st.multiselect() — Choose Many Options ✅✅

Like selectbox but the user can pick multiple items at once. Always returns a Python list.

python — multiselect_demo.py
import streamlit as st

st.title("Multiselect Demo")

skills = st.multiselect(
    "Select your QA skills:",
    ["Selenium", "Postman", "Jira", "SQL", "Jenkins", "Python", "TestNG"],
    default=["Selenium", "Postman"]  # Pre-selected items
)

# skills is ALWAYS a list
# Empty selection → skills = []
# 3 items picked → skills = ["Selenium", "SQL", "Python"]

if skills:
    st.write(f"You selected {len(skills)} skill(s):")
    for skill in skills:
        st.write(f"  ✅ {skill}")
else:
    st.warning("Please select at least one skill!")
📘 selectbox vs multiselect — Key difference
st.selectbox() → pick ONE → returns a single value (e.g. "Python")
st.multiselect() → pick MANY → returns a list (e.g. ["Python", "SQL"])
Lesson 09

st.checkbox() — On/Off Toggle ☑️

A simple tick box. Checked = True. Unchecked = False. Perfect for yes/no questions or to show/hide content.

python — checkbox_demo.py
import streamlit as st

st.title("Checkbox Demo")

# Unchecked by default (value defaults to False)
agree = st.checkbox("I agree to the terms and conditions")

if agree:
    st.success("✅ Thank you for agreeing!")
else:
    st.warning("⚠️ Please accept terms to continue.")

st.divider()

# Checked by default using value=True
show_details = st.checkbox("Show advanced details", value=True)

if show_details:
    st.write("🔍 Advanced details: Experience = 8+ years, Domain = Banking")

st.divider()

# Great use case: show/hide a section
show_raw = st.checkbox("Show raw data")
data = {"name": "Umakant", "exp": "8 years", "role": "QA Engineer"}
if show_raw:
    st.write(data)   # st.write can display dicts as a table!
Lesson 10

st.radio() — Pick ONE from a Visible List 📻

Shows all options visibly as radio buttons. User picks exactly one. Unlike selectbox (dropdown), all options are always visible on screen.

python — radio_demo.py
import streamlit as st

st.title("Radio Button Demo")

experience = st.radio(
    "Your experience level:",
    ["Beginner (0-2 yrs)", "Intermediate (2-5 yrs)", "Senior (5+ yrs)"],
    index=2   # Default: 3rd option (Senior) selected
)
st.write(f"Level: {experience}")

st.divider()

# horizontal=True puts all options side by side in one row
result = st.radio(
    "Test result:",
    ["✅ Pass", "❌ Fail", "⏸️ Blocked"],
    horizontal=True
)
st.write("Result marked:", result)

Lesson 11

st.date_input() and st.time_input() 📅⏰

Calendar and clock pickers. Click them to get a calendar popup or time selector. Returns Python datetime.date or datetime.time objects.

python — datetime_demo.py
import streamlit as st
import datetime

st.title("Date and Time Demo")

# Date picker — shows a calendar when clicked
dob = st.date_input(
    "Date of birth:",
    value=datetime.date(1990, 1, 1),    # Default date
    min_value=datetime.date(1950, 1, 1),
    max_value=datetime.date.today()
)
age_days = (datetime.date.today() - dob).days
st.write(f"You are {age_days // 365} years old!")

st.divider()

# Time picker
meeting_time = st.time_input(
    "Schedule standup meeting:",
    value=datetime.time(10, 0)   # Default: 10:00 AM
)
st.write(f"Meeting at: {meeting_time}")

st.divider()

# Date RANGE picker — pass a tuple as value!
date_range = st.date_input(
    "Test execution window:",
    value=(datetime.date.today(), datetime.date.today())
)
if len(date_range) == 2:
    st.write(f"Testing from {date_range[0]} to {date_range[1]}")
💡 QA Engineer Tip
As a QA engineer, date pickers are incredibly useful for: filtering test results by execution date, selecting sprint start/end dates, scheduling regression runs, and building test report dashboards. Skills you can use at work immediately!
Lesson 12

🏗️ Mini Project — Simple Calculator App

Let's combine everything from today into one real working app. This is exactly how Streamlit apps are built — put widgets together, add logic, show output.

python — calculator_app.py — copy and run this!
import streamlit as st

st.set_page_config(page_title="Calculator", page_icon="🧮")

st.title("🧮 Simple Calculator")
st.caption("Day 2 Mini Project — built with Streamlit widgets")
st.divider()

# Widget 1: First number
num1 = st.number_input("First number:", value=0.0, step=1.0)

# Widget 2: Pick operation — horizontal radio buttons
operation = st.radio(
    "Operation:",
    ["➕ Add", "➖ Subtract", "✖️ Multiply", "➗ Divide"],
    horizontal=True
)

# Widget 3: Second number
num2 = st.number_input("Second number:", value=0.0, step=1.0)

# Widget 4: Optional checkbox to show steps
show_steps = st.checkbox("Show calculation steps")

st.divider()

# Widget 5: Calculate button
if st.button("🔢 Calculate", type="primary"):

    if "Add" in operation:
        result, symbol = num1 + num2, "+"
    elif "Subtract" in operation:
        result, symbol = num1 - num2, "-"
    elif "Multiply" in operation:
        result, symbol = num1 * num2, "×"
    else:
        if num2 == 0:
            st.error("❌ Cannot divide by zero!")
            st.stop()   # Stop execution here!
        result, symbol = num1 / num2, "÷"

    if show_steps:
        st.info(f"Step: {num1} {symbol} {num2} = ?")

    st.success(f"✅ Answer: {num1} {symbol} {num2} = **{result}**")
💡 New: st.stop()
st.stop() immediately halts the entire script from running further. Like a return in a function, but for the whole page. Use it after showing an error to prevent invalid calculations from continuing!
