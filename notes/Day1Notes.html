Lesson 01

What is Streamlit? 🤔

Imagine you are a chef. You made an amazing dish (a Python data analysis or ML model). But your guests (your manager, client, or colleague) can't come into your kitchen. They want to see the dish on a nice plate, on the dining table.

🍽️ Analogy
Your Python script = The dish you cooked.
Streamlit = The dining table + the plate + the presentation.
It takes your Python code and puts it on a web page that anyone can see in a browser — without you learning any HTML, CSS, or JavaScript!

Streamlit is a free, open-source Python library that lets you build interactive web applications directly from your Python scripts. You write Python. Streamlit turns it into a website. That's it!

Who uses Streamlit?

  • Data Scientists who want to show their analysis
  • QA Engineers (like you!) who want dashboards for test results
  • ML Engineers who want to demo their models
  • Anyone who knows Python and wants a quick web app

Lesson 03

How Does Streamlit Work? ⚙️

This is the most important concept to understand before writing even one line of code. Let me explain it simply.

🎬 Analogy — The Movie Projector
Think of Streamlit like a movie projector. Every time someone interacts with the web page (clicks a button, moves a slider), the projector re-plays the entire movie from the beginning. That's exactly how Streamlit works — it re-runs your entire Python script from top to bottom every time something changes!

Here is the exact flow of what happens:

  1. You write a Python file, say app.py, with Streamlit commands.
  2. You run it using the command: streamlit run app.py
  3. Streamlit starts a small local web server on your computer.
  4. It opens your browser automatically at http://localhost:8501
  5. Your Python file runs top-to-bottom, and every st.something() call puts content on the page.
  6. When a user interacts (clicks, types), the whole script runs again from line 1.
⚠️ Important Rule to Remember
Streamlit reruns the entire script on every interaction. This means variables reset on every run unless you use special tools like st.session_state (we'll learn that on Day 5). For now, just know this behaviour!
🧠 Why is this useful?
Because you don't have to manually manage "what changes when button is clicked". Streamlit handles all that complexity. You just write your Python logic and it re-executes. Simple!
Lesson 04

Installation & Setup 🛠️

Let's get Streamlit running on your computer. This is super simple — just one command!

Step 1 — Make sure Python is installed (you already know Python, so you have this). Open your terminal / command prompt and type:

terminal / cmd 
python --version
# You should see something like: Python 3.9.7

Step 2 — Install Streamlit using pip (Python's package installer):

terminal / cmd 
pip install streamlit

Step 3 — Verify installation:

terminal / cmd 
streamlit --version
# You should see: Streamlit, version X.X.X

Step 4 — (Optional but recommended) Create a virtual environment to keep your projects clean:

terminal — optional best practice 
# Create virtual environment
python -m venv streamlit_env

# Activate it (Windows)
streamlit_env\Scripts\activate

# Activate it (Mac / Linux)
source streamlit_env/bin/activate

# Now install streamlit inside it
pip install streamlit
💡 What is a Virtual Environment?
Think of it like a separate room for each project. Each room has its own tools and libraries. So installing something for Project A doesn't break Project B. It's a professional best practice!
Lesson 05

Your First Streamlit App 🎉

Let's write your very first Streamlit app. Create a new file called app.py (or any name ending in .py) and type this:

python — app.py 
import streamlit as st

st.title("Hello, World! 🌍")
st.write("My name is Umakant Pandey.")
st.write("I am learning Streamlit today!")

Let's break it down line by line:

Line of CodeWhat it does (in plain English)
import streamlit as stWe bring the Streamlit library into our script and give it a short nickname st. Now instead of typing streamlit.title() every time, we just type st.title(). Saves typing!
st.title("Hello, World! 🌍")Puts a big bold heading on the page — like an H1 heading in a document.
st.write("My name is...")st.write() is the Swiss Army knife of Streamlit. It can display text, numbers, tables, dataframes, charts — almost anything! Here it shows a paragraph of text.
🧠 Note on "as st"
import streamlit as st — the as st part is just a nickname (alias). Programmers do this to save time. It's like calling your friend "Uma" instead of "Umakant" — both work, short is easier!
Lesson 06

Text Elements — Your Toolkit for Words 📝

Streamlit has many functions just for showing text. Think of them like different-sized stamps:

python — text_demo.py 
import streamlit as st

# ── TITLE: The biggest heading on the page ──────────────────
st.title("I am the Title — Big and Bold!")

# ── HEADER: Second level heading ────────────────────────────
st.header("I am a Header — Slightly Smaller")

# ── SUBHEADER: Third level heading ──────────────────────────
st.subheader("I am a Subheader — Even Smaller")

# ── WRITE: General purpose display ──────────────────────────
st.write("Hello! I can show text, numbers, even Python objects.")
st.write(42)                         # It shows the number 42
st.write([1, 2, 3])                  # It shows a list!

# ── TEXT: Plain text, no formatting ─────────────────────────
st.text("I am plain monospace text. Like a console output.")

# ── MARKDOWN: Write formatted text using Markdown syntax ────
st.markdown("**Bold text**, *italic text*, and `code text`")
st.markdown("## This is a Markdown Heading Level 2")
st.markdown("- Item 1\n- Item 2\n- Item 3")  # Bullet list!

# ── CAPTION: Small grey text, like footnotes ────────────────
st.caption("I am small grey text — good for footnotes or hints.")

# ── CODE: Shows code with syntax highlighting ───────────────
st.code("""
def greet(name):
    return f"Hello, {name}!"
""", language="python")

# ── DIVIDER: A horizontal line to separate sections ─────────
st.divider()   # Draws a horizontal line

# ── LATEX: For math formulas ─────────────────────────────────
st.latex(r"E = mc^2")  # Shows Einstein's formula beautifully!

Let's understand each one clearly:

FunctionThink of it as...When to use
st.title()The book's cover titlePage main heading, once per page
st.header()Chapter nameMajor section headings
st.subheader()Section within chapterSub-sections
st.write()A magic pen — writes anythingGeneral use, most flexible
st.text()Typewriter textShowing raw/fixed-width text
st.markdown()Word processor with formattingBold, italic, lists, links
st.caption()Photo caption or footnoteSmall explanatory text
st.code()Code editor viewDisplaying code snippets
st.divider()A ruled line in a notebookSeparating content visually
st.latex()Scientific journal formulaMath equations
🤔 What is Markdown?
Markdown is a simple way to format text using special characters. For example:
**bold** becomes bold
*italic* becomes italic
# Heading becomes a big heading
- item becomes a bullet point

st.markdown() understands these and converts them to beautiful formatted text on the page!
Lesson 07

Running the App & Hot Reload 🔥

Now let's run your app. Open your terminal, navigate to the folder where your app.py file is, and type:

terminal 
streamlit run app.py

What happens next?

  • Streamlit starts a server on your computer.
  • Your browser opens automatically at http://localhost:8501
  • You see your web app displayed in the browser!
🔥 Hot Reload — The Developer's Best Friend
Here is the amazing part: you don't need to stop and restart the app when you make changes!

When you save your app.py file after editing it, Streamlit will detect the change and show a message at the top of the browser: "Source file changed. Rerun?". Click Always Rerun once, and from that point on, every time you save your file, the browser automatically updates! It's like magic!

To stop the app: go back to the terminal and press Ctrl + C.

💡 Useful URLs when the app is running
Local URL: http://localhost:8501 — Open this on your own computer.
Network URL: http://192.168.x.x:8501 — Share this with anyone on your same WiFi network to show them your app live!
